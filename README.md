# KNN Diagnosis Tool for Embedding Comparison

A tool designed to **distinguish and compare multiple embeddings** of the same corpus generated by different language models (e.g., Longformer, BLOOM, GPT). By using **Recursive K-Nearest Neighbors (RKNN) graphs** and keyphrase analysis, this tool evaluates model performance and semantic clustering—even in the absence of ground truth data.

📄 **[Evaluating Foundation Models using Keyphrase Analysis without Ground Truth](report/KP_Evaluation_Report.pdf)**

---

## 📌 Overview
Selecting the best embedding model for a specific task is often hindered by a lack of labeled data. This tool bridges that gap by deriving features from the topology of **KNN and Recursive KNN graphs**. 

### The Core Hypothesis
The research suggests a strong correlation between language modeling metrics and embedding topology:
* **Perplexity ↔ Keyphrases**: As a model's **perplexity** decreases (better learning), the number of **common keyphrases** within a node's RKNN neighborhood increases.
* **Fertility ↔ Out-of-Distribution**: High **tokenization fertility** (sub-tokens per word) serves as a proxy for identifying Out-of-Distribution (OOD) data that the model struggles to represent.

---

## 🏗 Methodology & Workflow

### 1. Data Characterization
Before graph construction, I analyze the interaction between the data and the model tokenizer:
* **Tokenization Fertility**: Calculated as $\frac{\text{Number of Tokens}}{\text{Number of Words}}$. 
* High fertility indicates "difficult" or OOD text (e.g., medical jargon or technical reviews) where the model has to break words into many small sub-units.

### 2. Recursive KNN (RKNN) Graph
The tool constructs a graph where each document is a node. Edges are drawn based on **cosine similarity** or **L2 norm**. 
* **nHops**: Defines the depth of the neighborhood (exploring neighbors of neighbors).
* **Visual Signature**: A "decent" model produces neighborhoods with high keyphrase redundancy, indicating successful semantic mapping.

### 3. Keyphrase Extraction (TextRank)
Using the `pke` module, I extract keyphrases with the following constraints:
* **Grammar**: Focuses on ADJ, NOUN, PROPN, and VERB.
* **Refinement**: Phrases are filtered by the top 20% TF-IDF scores and top 10% frequency to eliminate noise.

## 📊 Experimental Insights

### Pre-training vs. Fine-tuning
I compared three versions of the **Longformer** model on the **AmazonCats-13k** dataset:
1. **Pre-trained**: General knowledge.
2. **Fine-tuned**: Optimized for the specific corpus.
3. **From-Scratch**: Trained specifically on the target domain.

**Key Finding**: Fine-tuned models show a significant "tightening" of the RKNN graph, where mean cosine similarity increases and the number of unique categories per neighborhood decreases.

### Out-of-Distribution (OOD) Analysis
The tool was tested across diverse domains to see how embedding features shift:
* **In-Distribution (ID)**: Amazon Reviews.
* **OOD**: MedQuad (Medical), FEVER (Fact Verification), and Emotional-Reddit.

| Metric | myn-Distribution | Out-of-Distribution |
| :--- | :--- | :--- |
| **Fertility** | Lower (Standard English) | Higher (Technical/Niche) |
| **Common Keyphrases** | High Overlap | Low Overlap (Sparse Graph) |
| **Cosine Similarity** | Concentrated/High | Distributed/Low |

---

## 🔍 Key Findings
* **Mean Cosine Similarity** is the most robust feature for distinguishing between different language models.
* **Community Detection**: Using `networkx.greedy_modularity_communities`, we found that better-performing models yield more distinct, dense clusters that correlate with actual document topics.
* **Semantic Integrity**: A "decent" model produces neighborhoods with high keyphrase redundancy, indicating that it has successfully mapped similar concepts to the same vector space.
